Category
Cross-Domain Controls - Cookie Protection via HttpOnly
Summary
Minimizes likelihood of session hijacking via cross site scripting by preventing JavaScript access to the cookies.
Details
In cross-site-scripting (XSS), attackers steal your browser cookies with JavaScript to obtain sensitive information such as sesion ID. However, when the set-cookie response contains the HTTPOnly attribute for a cookie, the browser disallows JavaScript access to the cookie. This defensive technique makes automated session hijacking attacks via XSS much more difficult.

HTTPOnly support is dependent on the browser. Most modern browsers do implement HTTPOnly.

Do It Yourself

For all instances where cookies are set, add the HTTPOnly attribute. So the response looks like: Set-Cookie: cookie1=foo; HttpOnly; cookie2=bar; HttpOnly;. Consult this page to see how to set-cookie on your web server.

If you are using Django, make sure you have SESSION_COOKIE_HTTPONLY = True in settings.py, since the default is unfortunately false.

Mozilla's Playdoh automatically have Set-Cookie: HttpOnly, ensure that you have commonware.response.cookies in your settings.py's installed apps.
