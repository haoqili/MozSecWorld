Summary
When sanitizing user input, MozSecWorld renders white-listed mark-up HTML tags and escapes the rest of the tags, including the potentially dangerous “<script>” tag.
Details

MozSecWorld uses an HTML sanitizing library [Bleach], which only allows certain safe html tags, such as “<b>” or “<i>”, to be rendered into html while the rest of the text are escaped. This way, users can still have some freedom on creating rich, fancier-looking text.

With Bleach, even if a malicious users injects XSS script into a webpage, the script is not going to be executed.
