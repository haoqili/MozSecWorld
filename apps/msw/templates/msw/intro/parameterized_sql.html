Input validation
Summary
Parameterizing SQL statements helps safely handle user data and prevent SQL injections.
Details
Incorporate user inputs directly into SQL queries may lead to [SQL injections] when a malicious user carefully crafts an input that causes the SQL parser to parse the query into something unintended.

Parameterized queries provides a mechanism for the parser to establish a clear differentiation between user submitted data and SQL syntax.

Do It Yourself

This page has examples on how to do SQL statement parameterization in different languages.

If you use Django, you don't need to write SQL queries yourself. But if you are in a situation where you do need to work with SQL directly, Django's raw() performs parameterized SQL statements for you. You cannot use raw() to run update, insert, or delete. So use cursor() instead, even though cursor() does not use parameterized SQL statementes, it escapes user inputs (not preferred)..
